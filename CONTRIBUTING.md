# Contributing Guidelines

Thank you for your interest in contributing to this project.

## Scope of Contributions

The following types of contributions are acceptable:

- Bug fixes
- Improvements to code quality and structure
- Documentation improvements
- Minor performance optimizations

The following are generally **out of scope**:

- Large architectural changes without prior discussion
- Feature requests that significantly change the project direction
- Breaking API changes
- Experimental or unreviewed dependencies

## Getting Started

1. Fork the repository
2. Create a new branch from `main`
3. Make your changes with clear, focused commits
4. Ensure the project builds and runs successfully using Docker Compose
5. Submit a pull request with a clear description of the changes

## Coding Standards

- Follow existing code style and conventions
- Keep changes minimal and focused
- Write code that is readable and maintainable

If applicable:
- Update documentation
- Add or update tests
- Ensure no existing functionality is broken

## Commit Messages

Use clear and descriptive commit messages.

## Pull Request Guidelines

- One logical change per pull request
- Describe **what** was changed and **why**
- Reference related issues if applicable
- Ensure the pull request can be reviewed independently

## Code Review

All contributions are subject to review by the project author.
